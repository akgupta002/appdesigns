Form elements are mostly wrapped with .form-group element. It can receive classes .has-error and .is-correct depending on the state. .form-control-indicator element can be used to emphasize the state.

Requires forms.js.

Checkboxes have a wrapper .checkbox and radios have .radio, which assist on the special styling of the elements.

The attribute data-checked="partially" is used for the custom checkbox used in the alerts screen.

Toggle buttons are different kind of checkboxes, which visualize the state of the input more. They have a wrapper .toggle and label .toggle-btn.

Number selector input has class name .input-number-selector and a placeholder and initial value attributes.

Date / Time picker input needs the attribute data-format to tell the plugin the desired format of the date.

Requires bootstrap-datetimepicker.min.js and needs to be initialized in the code with a reference to the ID of the wrapper:

$('#datetimepicker1').datetimepicker({
  language: 'en',
  pick12HourFormat: true
});

Buttons are normally button or a elements, but basically any element can be styled to look like a button with .btn class.

Buttons have 3 main themes, .btn-primary, .btn-secondary, .btn-alert. In addition .btn-tertiary is used on the dealer screens.
